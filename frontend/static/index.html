Upload Document

Hybrid mode uploads the file to Azure Blob, runs Azure AI Search skillset enrichment, and then merges those outputs back into the app-managed retrieval corpus.

Pipeline stages: Uploaded → Parsing → OCR/Layout extraction → Cleanup → Chunking → Metadata enrichment → Embedding/index preparation → Publishing → Ready for chat.

In hybrid mode, the source document is also uploaded to Azure Blob and passed through Azure AI Search skillsets so summaries, keyword hints, image descriptions, and other enrichment outputs can be merged into the retrieval corpus.

The random research action selects a meaningful topic pack and generates a long-form PDF above the Document Intelligence hard page limit so the backend must segment it before extraction.

The futures report action creates a 500+ page research-style PDF with ten sections and embedded diagrams to test large-document, skillset, and figure handling.

The construction report action creates a 500+ page construction-industry corpus with architecture diagrams and figure-ready pages for the pipeline.
